上下文与显存 这些参数共同决定可用上下文、启动显存和并发能力。
本地 Claude 推荐单人聊天用 4；冲 128K/192K 长上下文时用 1-2。
KV cacheFP8 最省显存，长上下文优先用 FP8；auto 通常会按模型 dtype 使用更大缓存。
OffloadCPU/KV offload 可以换更长上下文，但会牺牲速度，先从 4-8 GB 小步尝试。
仅语言模式不用图片/视频时可开启，减少多模态 encoder 和缓存开销。
输入模型后会估算对应上下文长度的显存占用。
模型名里包含 7B、14B、27B 这类规格时估算最准确；长上下文主要增加 KV cache 显存。
GGUF 启动参数 vLLM 的 GGUF 支持仍偏实验；推荐只用单文件 GGUF。
选择 GGUF 时会直接把 .gguf 文件传给 vLLM，并忽略 AWQ/GPTQ/Compressed 这类额外权重量化参数。
Open WebUI 建议用 qwen3 / deepseek_r1 这类 parser 输出结构化 reasoning；Claude Code 或 Claude Cowork 通过 OpenAI 兼容桥接时，要看桥接器是否转发 reasoning_content，不支持时可关闭。
工具调用桥接 Claude 会把工具 schema 发给管理器，管理器再转成 vLLM/OpenAI tools。
Qwen3.6 / Qwen3-Coder 推荐 qwen3_coder；DeepSeek 推荐 deepseek_v3；Qwen2.5 常用 hermes。
Claude 上下文自动压缩 仅作用于 Claude 兼容桥；触发后保留最近原文，把更旧内容压成结构化摘要。
默认 90% 触发、20% 最近原文、20% 摘要预算；错误、路径、目标、硬性要求和工具结果会优先保护。
仅本机访问会绑定 127.0.0.1；局域网访问会绑定本机 IPv4 地址。设置 API Key 后，客户端需以 Bearer Token 携带。
GPU 设备 勾选要暴露给 vLLM 的显卡；单卡机器会自动选中 GPU 0。

在线模型

从 Hugging Face Hub 实时获取，适合找最新、热门、蒸馏和去审查模型。

在线列表按公开模型元数据返回；gated 模型下载前需要配置对应 token。

下载模型

粘贴 Hugging Face 或 ModelScope 的模型页面链接，可自动填入模型 ID、保存名和下载来源。
选择预设后仍可手动改模型 ID。 规格越大效果通常越好但更吃显存；量化精度越低越省显存，可能损失一点质量。
Hugging Face 通用性最好；ModelScope 在国内网络环境通常更顺手。
填入模型 ID 后会预估下载体积。
